âœ… **Using Prepared Statements** (Prevents SQL Injection)  
- **âŒ Bad Practice (Vulnerable to SQL Injection)**  
  ```java

  ResultSet rs = stmt.executeQuery("SELECT * FROM users WHERE username = '" + userInput + "'");
  ```
- **âœ… Best Practice (Using PreparedStatement)**  
  ```java

  ResultSet rs = pstmt.executeQuery();
  ```

âœ… **Using Transactions in JDBC**  
- **By default, each SQL statement is auto-committed.** To **group multiple queries in one transaction**, use:
  ```java
  con.setAutoCommit(false);
  try {
      stmt.executeUpdate("INSERT INTO orders ...");
      stmt.executeUpdate("UPDATE inventory ...");
      con.commit();  // âœ… Commit transaction
  } catch (SQLException e) {
      con.rollback();  // âŒ Rollback if any failure
      e.printStackTrace();
  } finally {
      con.setAutoCommit(true);
  }
  ```

âœ… **Batch Processing for Performance Optimization**  
Instead of executing multiple queries one by one:
  ```java
  stmt.addBatch("INSERT INTO employees VALUES (1, 'Alice')");
  stmt.addBatch("INSERT INTO employees VALUES (2, 'Bob')");
  stmt.executeBatch(); // âœ… Executes all at once, improving performance
  ```

âœ… **Using Connection Pooling (HikariCP, C3P0, Apache DBCP)**  
- Instead of creating a new connection for each request (**slow** ðŸ¢), **use a connection pool**:
  ```java

  config.setJdbcUrl("jdbc:postgresql://localhost:5432/testdb");
  config.setUsername("postgres");
  config.setPassword("password");
  HikariDataSource ds = new HikariDataSource(config);

  Connection con = ds.getConnection(); // âœ… Uses pooled connection
  ```

âœ… **Reading Database Metadata**  
- Get **table names, column types, indexes** dynamically:
